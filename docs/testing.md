# Testing

`npm run test` covers pure domain rules for referral transitions, room availability, temporal occupancy overlap, overdue tasks, integer-money invoice totals and remaining balances, CSV escaping/formula protection, and public/protected form validation.

The billing smoke workflow also verifies an authenticated claim update, invoice creation, issue transition, partial payment, final payment, and organisation-scoped cleanup against the local database. The reports smoke workflow verifies scoped search, all four CSV report headers, export audit events, and a denied finance export for a support lead. The placement/admin smoke workflow verifies room-overlap rejection, move-out, invitation, deactivation, non-admin denial, and exact cleanup. The content smoke workflow verifies live settings, service/news public projections, audited mutations, content-write denial, and restoration of seeded values. Governance smoke coverage verifies dry-run retention summaries, future per-record retention dates, legal-hold/open-request exclusions, scheduler-secret protection, and no deletion side effects. Security tests cover bounded throttling, forwarded-client parsing, and production configuration readiness.

The current Playwright smoke suite covers public referral navigation and logged-out CRM protection. Expand it against fictional seeded data to cover public referral submission, invalid errors, staff login, referral acknowledgement and assignment, referral-to-placement transaction, support plan and case note creation, restricted-note denial, incident closure permissions, Housing Benefit updates, news publishing, draft visibility, confidential property projection, and audited CSV export.

Before production, add service/property scope fixtures and run the suite against the same database engine and storage/email adapters used in the deployment. Run `npm run check:production-config` with the deployment environment and confirm `/health` returns HTTP 200. Verify the HTTP rate-limit adapter receives only hashed keys, fails closed when unavailable, and is protected by its token. Verify production email fails closed when SMTP is absent. Verify local and S3-compatible document writes, reads, deletes, and private-bucket settings with a provider test bucket. Verify recovery codes are displayed only after enrolment, accepted once, and never returned by subsequent login responses. Verify document uploads are scan-before-write, rejected when scanning is unavailable in production, and blocked when integrity hashes do not match.
